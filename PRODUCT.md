# Tech Reader 产品文档

## 一、产品定位

**Tech Reader** 是一个面向开发者的 Firefox 插件，核心目标是：

> 帮助开发者摆脱对翻译工具的依赖，真正培养英文技术文档的阅读能力。

### 和普通翻译插件的本质区别

| | 普通翻译插件 | Tech Reader |
|---|---|---|
| 核心行为 | 把英文替换成中文 | 帮你理解英文的结构和意思 |
| 使用越久 | 越依赖 | 越不需要它 |
| 输出形式 | 中文译文 | 句子结构 + 语义解释 |
| 目标 | 看懂这篇文章 | 提升读英文的能力 |

**一句话：用得越多，越不需要它。**

---

## 二、核心阅读理念

Tech Reader 的所有功能都基于以下两个原则：

### 原则一：语义块优先，不逐词翻译

技术英语大量使用固定短语和抽象表达，必须按语义块理解：

```
look to      → 借助 / 依靠（不是"看"+"向"）
take on      → 承接 / 接手
build on     → 基于……继续发展
eat up       → 占用（内存/时间）
shoulder     → 承担（责任）
open up      → 带来 / 开启
```

### 原则二：树状阅读，不线性阅读

任何长句本质都是一棵树，不是一条线：

```
ROOT（主干：谁做什么）
├── 修饰主语
├── 修饰谓语（怎么做）
├── 目的
├── 条件
└── 举例 / 补充
```

阅读顺序：先抓主干，再看分支，而不是从左到右逐词读。

## 三、功能说明

### 功能一：句型分析（选句触发）

**使用方式：** 在任意英文页面选中一句话，点击出现的「分析」按钮。

**面板分三层，用户主动控制展开节奏：**

**第一层 — 句子骨架**（默认展开）

剥去所有修饰成分，只留核心主谓宾：

```
主语  React
谓语  updates
宾语  the UI
```

**第二层 — AST 树形分析**（点击展开）

将句子解析成树状结构：

```
ROOT  React updates the UI
├── 修饰主语  that manages state
├── 目的     to reflect the latest data
└── 条件     when props or state change
句型  relative clause
```

同时识别句中的固定短语：

```
rely on  →  依靠
```

**第三层 — 用自己的话说**（点击展开）

不是直译，而是：如果你完全理解了这句话，你会怎么跟同事解释？

```
React 负责更新界面，一旦 props 或 state 变了，
它就会重新渲染，把最新数据展示出来。
```

---

### 功能二：Hover 查词

**使用方式：** 鼠标悬停在任意单词上，停留 0.6 秒自动触发。

**展示内容：**
- 词性
- 英文解释（不给中文翻译）
- 例句
- 如果查过，显示「你已查过 N 次」

**为什么给英文解释而不是中文翻译？**

目标是建立「英文 → 概念」的直接连接，而不是「英文 → 中文 → 概念」的中转。

---

### 功能三：本地词库

每次查词自动记录到本地 IndexedDB：

- 单词、词性、英文解释
- 首次查询时间、最近查询时间
- 查询次数（用于衰减提醒）

数据**完全存储在本地**，不上传任何服务器。

---

### 功能四：自定义 AI 接口

支持所有兼容 OpenAI 格式的服务：

| 服务 | Base URL |
|---|---|
| OpenAI | https://api.openai.com/v1 |
| DeepSeek | https://api.deepseek.com/v1 |
| 通义千问 | https://dashscope.aliyuncs.com/compatible-mode/v1 |
| 月之暗面 | https://api.moonshot.cn/v1 |

推荐模型：`deepseek-chat`（效果好，价格低）

## 四、技术架构

```
tech-reader/
├── manifest.json        Firefox MV2 插件配置
├── background.js        AI 接口调用（绕过页面 CSP）
├── content.js           页面交互：选词监听、面板注入、hover 查词
├── panel/
│   └── panel.css        分析面板 + tooltip 样式
├── settings/
│   ├── settings.html    设置页
│   └── settings.js      API 配置读写
└── storage.js           IndexedDB 词库封装
```

**技术选型：**
- Firefox WebExtension MV2（persistent background）
- IndexedDB 本地存储
- OpenAI Chat Completions API（兼容格式）
- 原生 JS + CSS，无任何框架依赖

## 六、目标用户

- 有一定英文基础，但阅读技术文档还需要借助翻译工具的开发者
- 意识到自己对翻译插件过度依赖、想改变这个习惯的人
- 想在阅读过程中顺带积累技术词汇的开发者

**不适合：** 完全不想学英文、只想快速获取信息的用户（那用沉浸式翻译就好了）
